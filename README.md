# Chain Finder — Cosmos Validator & Chain Monitor Bot

**Chain Finder** is an automated monitoring bot for the Cosmos ecosystem. It tracks specific Twitter accounts for new posts (like/quote/retweet with human-like behavior) and searches for new Cosmos chains via Twitter keywords and GitHub PRs. All notifications are delivered to a Telegram channel.

---

## Features

### Function 1 — Account Timeline Monitor
- Monitors a configurable list of Twitter accounts (e.g., `@_atomone`, `@Hippo_Protocol`).
- Fetches new posts (up to 7 days old).
- Performs **Like + Quote** (quote generated by LLM via Knight) or falls back to **Like + Retweet**.
- FIFO queue: processes 1–4 posts per run (random), leftover posts wait for the next cycle.

### Function 2 — Chain Finder (Twitter + GitHub)
- Searches Twitter for user-defined keywords (e.g., `"new validator"`, `"cosmos sdk"`).
- Monitors GitHub PRs from 4 Cosmos chain registries:
  - `chainapsis/keplr-chain-registry`
  - `cosmos/chain-registry`
  - `ping-pub/mainnet`
  - `ping-pub/testnet`
- Detects PRs adding `chain.json` or `assetlist.json`.
- Sends notifications only (no actions).

### Smart Scheduling
- **Active hours:** 08.00 – 21.00 WIB — actions and real-time notifications.
- **Quiet hours:** 21.00 – 05.00 WIB — still fetches data, but **actions are paused** and notifications are **held** until the daily digest at **05.05 WIB**.
- Digest combines both timeline and chain-finder notifications into one message.

---

## How It Works

- **Playwright** with browser cookies is used for Twitter automation (no OAuth required).
- **LLM integration:** quote generation uses Knight model via 9router (`http://127.0.0.1:20128/v1`).
- **Retry mechanism:** 3 attempts with exponential backoff for fetch and actions.
- **Deduplication:** seen tweets stored in `timeline_seen.json` and `seen_tweets_chain.json`.
- **Queue:** `timeline_queue.json` holds pending actions (max 50).

## Installation & Setup

### 1. Clone the Repository
```bash
git clone https://github.com/Fullbuster0/chain-finder.git
cd chain-finder
```

### 2. Install Dependencies
Ensure Python 3.11+ and a virtual environment are available.
```bash
pip install playwright requests
playwright install chromium
```

### 3. Configure Twitter Cookies
Export cookies from your logged-in Twitter browser session as a **Netscape format** file (e.g., using browser extensions like "Export Cookies"). Place the file as `x_cookies.txt` in the project root. The bot uses these cookies for authentication.

### 4. Configure Accounts & Keywords
Edit the configuration variables directly in the scripts:

- **`timeline_monitor.py`** — `TARGET_ACCOUNTS` list (add/remove Twitter usernames).
- **`chain_finder_x.py`** — `KEYWORDS` list (add/remove search terms).

No other code changes are required.

### 5. Set Up LLM Provider (Optional)
Quote generation requires a running LLM endpoint. By default, the bot expects `Knight` at `http://127.0.0.1:20128/v1/chat/completions`. If you want to use a different provider, modify `LLM_URL` and `LLM_MODEL` in `timeline_monitor.py`.

### 6. Run the Bot
**Manual run:**
```bash
python3 timeline_monitor.py normal      # fetch timeline and process actions
python3 timeline_monitor.py digest      # send daily digest (pending notifications)
python3 chain_finder_x.py               # run Twitter keyword search
python3 chain_finder_github.py          # run GitHub PR monitor
```

**Set up cron jobs** (example schedules):
- Timeline monitor: every 2 hours (`0 */2 * * *`)
- Chain finder X: every 1 hour (`0 * * * *`)
- GitHub chain finder: every 1 hour (`0 * * * *`)
- Digest: daily at 05:05 WIB (`5 5 * * *`)

## Configuration Variables

| Script | Variable | Purpose |
|--------|----------|---------|
| `timeline_monitor.py` | `TARGET_ACCOUNTS` | List of Twitter usernames to monitor |
| `timeline_monitor.py` | `MAX_AGE_DAYS` | Only process posts newer than this (default 7) |
| `timeline_monitor.py` | `ACTIONS_PER_RUN_MIN/MAX` | Number of actions per run (random between 1–4) |
| `timeline_monitor.py` | `ACTIVE_HOUR_START/END` | Active hours in WIB (8–21) |
| `timeline_monitor.py` | `LLM_URL`, `LLM_MODEL` | Endpoint for quote generation |
| `chain_finder_x.py` | `KEYWORDS` | List of search keywords |
| `chain_finder_github.py` | (hardcoded repo list) | 4 Cosmos chain registries |

## Troubleshooting

### Playwright Issues
- Ensure Chromium is installed: `playwright install chromium`
- If cookies fail, re‑export fresh cookies from your browser.

### LLM Quote Generation Fails
- Check that the LLM endpoint is reachable.
- The bot falls back to retweet if quote fails.

### Rate Limiting
- The bot uses random delays (30–120s) between actions and respects quiet hours to avoid detection.
